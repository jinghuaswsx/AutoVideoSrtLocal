人声分离 + 响度匹配

总开关

关闭时多语种 / 全能视频翻译走旧逻辑：TTS 直接替换原音轨、不做响度匹配、不混背景音。 打开后会在 ASR 步骤之后调用本机 GPU API 做人声分离（同步阻塞 ~10-60s 视频长度）， 并在 TTS 完成后做 EBU R128 二阶段响度归一化到原视频人声基准 L₀（±3% 收敛）。

API 接入

上游服务跑在本机 AudioSeparator（默认 http://127.0.0.1:83），接口协议参考 nomadkaraoke/python-audio-separator。 留空时等同于关闭总开关。

服务端从这里探活 {api_url}/health，并发 multipart 上传到 {api_url}/separate/download。

vocal_balanced 是上游服务的默认推荐，整体最均衡。其它选项见 /presets 接口。

覆盖 GPU 排队 + 推理 + ZIP 下载的全过程。上游 API 文档建议至少 300s。 超过此阈值客户端按"已超时"降级，主任务继续走旧逻辑。

合成最终硬字幕 mp4 时，分离出来的 background 轨与 TTS 主轨 amix 的音量乘数。 默认 0.6（约 -4.4 dB），让 BGM/环境音不抢戏但仍可闻。CapCut 工程包导出走另一条路径， 保留独立 ambience 音轨让用户在剪映里单独调音，**不**受这里影响。
