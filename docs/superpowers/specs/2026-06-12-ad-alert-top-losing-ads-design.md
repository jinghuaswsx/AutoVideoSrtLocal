# 广告预警卡片级亏损 AD 预览优化设计

## 背景

当前预警列表卡片只展示商品×语言的汇总 ROAS，运营人员必须在卡片上看到`ROAS 0.87`，**猜测**可能有某条 AD 在亏钱，然后点开详情弹窗才看到具体哪些 AD 在亏。核心信息埋得太深。

## 目标

在每张预警卡片上直接展示该语言下**亏损最严重的 2-3 条 AD**，并支持直接在卡片级调用 AI 评估，减少操作层级。

## 改动范围

### 1. `AlertItem` 数据模型新增 `top_losing_ads` 字段

每个 AlertItem 附加上该语言下按 ROAS 升序排列的最亏 2-3 条 AD。

### 2. 卡片 UI 新增亏损 AD 预览行

```
┌─────────────────────────────────────────────────────────┐
│ 🇩🇪 DE · 商品名称 ABC123                         严重   │
│ ROAS: 0.87 | 消耗 $1,240 | 运行 23天 | 预亏 -$165    │
│ ──────────────────────────────────────────────────────  │
│ 🔴 US · ABC123_en_01 · ROAS 0.72 · 亏 -$248           │
│ 🟠 AU · ABC123_au_01 · ROAS 1.12 · 亏 -$36            │
│ [🤖 AI 评估这组广告]                                    │
│ ⚠️ 建议关停：ROAS 低于 1.0 且已运行超过 7 天...       │
└─────────────────────────────────────────────────────────┘
```

### 3. 涉及文件

| 文件 | 改动 |
|------|------|
| `appcore/ad_alerts.py` | `AlertItem` 新增 `top_losing_ads`；`get_alerts()` 附加上；新增 `_get_top_losing_ads()` |
| `web/routes/ad_alerts.py` | `_alert_item_to_dict()` 序列化新增字段 |
| `web/templates/ad_alerts.html` | 卡片模板中渲染亏损 AD 预览 + 每卡一个 AI 评估按钮 |
