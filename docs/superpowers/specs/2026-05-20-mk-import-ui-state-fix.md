# 明空入库后前端状态同步修复

## 背景

明空素材卡片点击「加入素材库」成功后，当前页面仍可能显示「加入素材库」。原因是前端只更新触发请求的单个按钮，同时把 filename 加进已检查缓存；如果页面存在同素材重复卡片或后续渲染同 filename，批量检查会跳过它，按钮状态就不会再被修正。

入库进度弹窗里的「去素材管理」也需要直接进入素材管理搜索页，使用最终素材库产品 ID（带 `-rjc` 后缀）作为 `q` 参数，方便人工立刻定位刚入库的产品。

## 修复范围

1. 新增前端已入库 filename 集合，成功入库后按 filename 同步所有匹配的「加入素材库」按钮。
2. 批量检查遇到本页已知已入库 filename 时，直接应用已入库 UI 状态，不再因为已检查缓存跳过。
3. 弹窗「去素材管理」根据按钮上的 `data-mki-product-code` 或商品链接生成 `/medias/?q=<code-rjc>`。
4. 不改后端入库接口契约，不新增数据库字段。

## 2026-05-20 文件名显示回归

明空视频入库到素材管理后，视频素材卡片的名称必须使用明空原始视频文件名，例如 `2026.04.09-物理综合实验DIY-混剪-苏齐齐.mp4`，不能写成产品中文名。产品名仍只用于 `media_products.name`。
