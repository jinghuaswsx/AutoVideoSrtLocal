# 素材管理一键翻译编排与任务管理设计

- 日期：2026-04-22
- 作者：Codex + 用户协作确认
- 状态：Design
- 所属模块：`medias` / `bulk_translate` / `multi_translate`

---

## 1. 背景与目标

当前素材管理页已经有“原始视频翻译”弹窗，但能力仅覆盖原始视频到多语种视频的单一路径，且任务管理仍偏向通用 `bulk_translate` 批次视角，不适合运营按产品统一盯进度。

本次目标是把素材管理里的“翻译”升级为**产品级一键翻译编排器**：

- 一次发起同时覆盖 4 类内容：
  - 文案翻译
  - 商品详情图翻译
  - 视频封面翻译
  - 视频翻译
- 创建弹窗只负责配置和发起任务，不承担任务管理
- 任务管理改成**产品优先**视角，从素材管理列表可以直接进入当前产品的翻译任务页
- 视频翻译流程默认全自动推进，但**统一停在选声音步骤**等待人工介入
- 服务器重启或其他中断后，系统启动时必须把相关任务标记为 `interrupted`，**不自动续跑**
- 字幕位置预览能力抽成共享组件，后续复用到多语言视频翻译页

## 2. 已确认的产品决策

### 2.1 入口与页面结构

- 素材管理列表页每个产品行保留两个入口：
  - `翻译`
  - `翻译任务管理`
- 点击 `翻译`：
  - 打开创建任务弹窗
  - 弹窗只做创建任务
  - 弹窗里保留一个次级入口 `去任务管理`
- 点击 `翻译任务管理`：
  - 直接进入当前产品的翻译任务详情页
  - 不先进入全局任务列表

### 2.2 创建弹窗结构

- 原始视频选择：默认全选当前产品的英文原始视频
- 翻译内容：4 个复选项，默认全部勾选
- 目标语种：默认勾选“当前还没有视频成品”的语种
- 视频翻译必要配置：只保留
  - 字体
  - 字号
  - 字幕位置

### 2.3 字幕预览交互

- 预览区使用单个 `270x480` 视频播放框
- 默认加载当前产品第一个“英文且带字幕”的视频
- 字幕预览层直接叠加在真实视频播放框上
- 用户拖动的是字幕叠加层，不再使用左右两个分离的模拟框
- 调整 `字体 / 字号 / 字幕位置` 时必须立刻看到效果

### 2.4 任务管理页结构

- 默认按“产品”分组
- 当前产品页内部按“批次”展开
- 每个批次下再按 4 类任务展示
- 需要明确区分 3 种状态：
  - 失败
  - 中断
  - 正常等待选声音

## 3. 推荐方案与总体架构

本次采用**方案 A**：在现有 `bulk_translate` 基础上升级为统一父任务编排器。

### 3.1 为什么选这个方案

现有代码已经具备以下骨架：

- [appcore/bulk_translate_plan.py](../../../appcore/bulk_translate_plan.py)
- [appcore/bulk_translate_runtime.py](../../../appcore/bulk_translate_runtime.py)
- [web/routes/bulk_translate.py](../../../web/routes/bulk_translate.py)
- [web/routes/medias.py](../../../web/routes/medias.py)

这些能力已包含：

- 父任务概念
- 计划展开
- 子任务调度
- 状态保存
- 失败后重试接口
- 审计字段与成本统计
- 视频参数快照

因此本次不新造第二套父任务系统，而是在现有 `bulk_translate` 之上做三类升级：

1. 从“批次导向”升级为“产品优先展示”
2. 从“同步串行执行器”升级为“受控派发 + 异步观察回填”的编排器
3. 从“通用 bulk_translate”升级为“素材管理专用的一键翻译任务中心”

### 3.2 核心组件分层

- 创建入口层
  - `web/templates/medias_list.html`
  - `web/static/medias.js`
- 任务编排层
  - `appcore/bulk_translate_plan.py`
  - `appcore/bulk_translate_runtime.py`
- 业务执行层
  - 文案：现有多语言标题/文案翻译
  - 详情图：现有 `image_translate`
  - 视频封面：复用 `image_translate` 作为“封面批量翻译”
  - 视频：现有 `multi_translate` / `translate_lab` 流程
- 任务管理展示层
  - 新增产品级任务页路由与模板
- 共享字幕预览层
  - 新增独立 partial + JS，供素材创建弹窗和多语言视频页复用

## 4. 数据模型与任务关系

### 4.1 任务关系

- 一个产品可有多次翻译批次
- 每次点一次“创建翻译任务”，生成 1 个父任务（批次）
- 每个父任务下有 4 类业务子任务：
  - `copywriting`
  - `detail_images`
  - `video_covers`
  - `videos`
- 每类任务再按语言拆分
- 视频翻译如果勾选多个原始视频，则继续拆到 `视频 × 语种`

### 4.2 父任务 state 的新增约束

在现有 `bulk_translate` state 基础上，补充或重定义：

- `product_id`
- `raw_source_ids`
- `content_types`
- `target_langs`
- `video_params_snapshot`
- `dispatch_state`
  - 记录 4 类任务当前的派发游标与下一次允许创建时间
- `child_index`
  - 建立“计划项 -> 子任务 id -> 当前状态”的索引
- `rollup`
  - 页面直接消费的聚合态
- `audit_events`
- `result_sync_state`
  - 已回填与未回填结果标记

### 4.3 素材回填关联字段

本次继续使用已有迁移中的字段体系：

- `source_ref_id`
- `bulk_task_id`
- `auto_translated`
- `manually_edited_at`

要求：

- 所有自动结果都必须带上 `bulk_task_id`
- 手工修改后不清空关联，只补 `manually_edited_at`

## 5. 创建弹窗设计

### 5.1 区块结构

从上到下分 5 个区块：

1. 选择原始视频
2. 翻译内容
3. 目标语种
4. 视频翻译配置
5. 底部动作区

### 5.2 默认行为

- 原始视频：默认全选
- 翻译内容：4 项默认全选
- 目标语种：默认勾选“当前缺少视频成品”的语种
- 视频配置：
  - 优先带出当前产品最近一次保存的配置
  - 没有则走系统默认

### 5.3 资源不足时的禁用规则

- 没有英文原始视频：
  - `视频封面翻译`、`视频翻译` 可见但禁用
- 没有英文详情图：
  - `商品详情图翻译` 可见但禁用
- 没有英文文案：
  - `文案翻译` 可见但禁用

### 5.4 字幕预览面板

- 使用单个 `270x480` 视频框
- 默认视频来源：
  - 优先当前产品第一个“英文且带字幕”的视频
  - 否则退回第一个英文原始视频
- 拖动字幕叠加层时实时更新 `subtitle_position_y`
- 保存时将以下值冻结为本次批次快照：
  - `subtitle_font`
  - `subtitle_size`
  - `subtitle_position_y`

## 6. 四类任务编排策略

## 6.1 总原则

父任务拆为两阶段职责：

- `dispatch loop`
  - 控制“什么时候创建哪个子任务”
- `watch loop`
  - 观察子任务状态、回填结果、更新父任务聚合状态

原因：

- 需要按固定节奏创建任务
- 需要允许图片任务内部并行
- 需要允许视频任务停在选声音
- 需要在子任务完成时即时回填业务结果

### 6.2 文案翻译

- 优先执行
- 复用现有多语言标题/文案翻译能力
- 拆分方式：
  - `英文文案 × 目标语种`
- 成功后立即回填目标语种文案

### 6.3 商品详情图翻译

- 拆分方式：
  - 每个目标语种 1 个图片翻译子任务
  - 每个子任务内包含当前产品全部英文详情图
- 派发节奏：
  - 相邻两个语种任务创建间隔 30 秒
- 执行方式：
  - 子任务内部继续使用现有图片并行翻译
- 完成后：
  - 立即回填该语种详情图集合

### 6.4 视频封面翻译

- 定义为“所选原始视频封面”的翻译，不混入商品主图
- 拆分方式：
  - 每个目标语种 1 个封面翻译子任务
  - 任务内包含所有选中的原始视频封面
- 完成后：
  - 立即回填该语种的视频封面结果

### 6.5 视频翻译

- 拆分方式：
  - `原始视频 × 目标语种`
- 派发节奏：
  - 相邻两个视频翻译子任务创建间隔 2 分钟
- 流程：
  - 自动跑到 `voice_match`
  - 在“选声音”步骤停止
  - 状态记为 `awaiting_voice`
- 用户确认声音后：
  - 自动恢复后续步骤
  - 完成后回填视频成品

## 7. 状态模型与恢复机制

### 7.1 父任务状态

建议父任务状态收敛为：

- `planning`
- `running`
- `waiting_manual`
- `failed`
- `interrupted`
- `done`

定义：

- `waiting_manual`
  - 代表没有真实失败，但存在视频任务停在选声音

### 7.2 子任务状态

- `pending`
- `dispatching`
- `running`
- `awaiting_voice`
- `syncing_result`
- `done`
- `failed`
- `interrupted`
- `skipped`

### 7.3 页面动作与状态映射

- `failed` -> `重新启动`
- `interrupted` -> `从中断点继续`
- `awaiting_voice` -> `去选声音`
- `done` -> `查看结果`
- `running` -> 仅展示实时进度

### 7.4 失败重跑原则

不做整批重来，只从失败点继续：

- 文案：仅重跑失败语种
- 详情图：仅重建失败语种图片任务
- 封面：仅重建失败语种封面任务
- 视频：仅重建失败的视频语种任务
- `awaiting_voice` 不算失败，不进入“重跑”通道

### 7.5 启动时中断标记

现有 `bulk_translate` 规则中“不扫描、不对账”与本次需求冲突，本次升级为：

- 系统启动时允许扫描 `bulk_translate`
- 扫描目的仅限：
  - 把处于 `dispatching / running / syncing_result` 的父任务和子任务标记为 `interrupted`
- 明确禁止：
  - 自动恢复执行
  - 自动补跑
  - 自动续到下一步

特例：

- `awaiting_voice` 不标为中断
- 因其本身是业务上故意暂停，不属于异常恢复场景

## 8. 结果回填规则

### 8.1 文案

- 成功后写入目标语种文案记录
- 写入自动翻译关联字段
- 手工修改后只补 `manually_edited_at`

### 8.2 详情图

- 子任务完成后立即回填该语种详情图集合
- 保留现有规则：
  - 不误删手动上传图片
  - 仅替换自动翻译产物

### 8.3 视频封面

- 子任务完成后回填“原始视频 × 语种”的封面结果
- 结果不混入商品主图

### 8.4 视频

- `awaiting_voice` 阶段仅保存过程状态，不生成最终成品
- 选完声音并完成后续流程后，再正式创建目标语种视频成品记录

## 13. 验收口径

当以下条件同时成立时，视为本设计达标：

- 素材管理列表页可以从单个产品发起 4 类翻译批次
- 创建弹窗默认勾选符合“缺失视频语种优先”的规则
- 视频字幕配置在单个真实视频框内所见即所得
- 产品级任务管理页能查看当前产品所有翻译批次
- 用户能清晰区分失败、中断、等待选声音
- 服务器重启后任务只被标记中断，不会自动续跑
- 视频任务统一在选声音步骤停下，之后只需人工选声音即可继续
