# 素材管理顶部工具区紧凑布局设计

- 日期：2026-05-10
- 范围：素材管理列表页 `/medias`
- 文档锚点：`AGENTS.md` 文档驱动代码、`web/static/CLAUDE.md` Ocean Blue 控件约束、`docs/superpowers/specs/2026-04-13-medias-management-design.md` 素材管理列表页、`docs/superpowers/specs/2026-05-01-mobile-ios-responsive-design.md` 移动端素材管理适配。

## 背景

素材管理页在移动端顶部操作区占用纵向空间过大：添加产品素材、下载自动换图工具、搜索框、两个筛选框和搜索按钮被挤压成多行，列表首屏内容被明显下推。

现有 `web/static/medias.js` 已经在关键词输入时触发 250ms 防抖实时搜索，点击「搜索」按钮不再是主要操作路径。

## 目标

1. 顶部两个蓝色主操作放在同一行：`添加产品素材` 与 `下载自动换图工具`。
2. 搜索与筛选放在第二行：搜索框、小秘云仓筛选、ROAS 数据筛选各占 1/3 宽度。
3. 隐藏文字搜索按钮；关键词输入仍实时出结果，按 Enter 仍同步地址栏查询参数并搜索。
4. 桌面端保留下载工具版本号和发布时间；移动端不让这些元信息额外占用一行，下载按钮通过 `title` 保留版本提示。

## 不做

- 不改素材列表 API、筛选语义、分页逻辑。
- 不新增路由和权限。
- 不改变「添加产品素材」或「下载自动换图工具」的行为。

## 验收

1. `/medias` 顶部主操作在移动端同一行显示两个蓝色按钮。
2. 搜索框和两个筛选框在同一行内按三等分排列。
3. 页面 HTML 不再渲染带「搜索」文字的按钮。
4. `kw` 输入框在没有 `searchBtn` 的情况下仍绑定 input/Enter 搜索事件。

## 2026-05-10 回归修正

移动端 `.oc-header-actions` 必须覆盖桌面 `flex:1 1 420px`，使用 `flex:0 0 auto`，否则纵向布局时该 flex-basis 会把 header 撑出大块空白，导致筛选区被推到首屏底部。
