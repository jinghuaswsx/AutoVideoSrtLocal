# API 账单日期范围选择设计

**日期：** 2026-04-24  
**分支：** `codex/ai-billing-date-range`  
**Worktree：** `.worktrees/codex-ai-billing-date-range`

## 目标

把 API 账单页面顶部的两个独立日期输入框合并成一个日期范围交互：

1. 用户点击一次日期范围控件即可打开面板。
2. 第一次点击日期选开始，第二次点击日期选结束。
3. 第二次选完立即提交筛选并刷新数据，不再需要点回车或确认。
4. 增加“今天 / 昨天 / 近七天 / 近一个月”四个快捷按钮，点击后立即写入 `from/to` 并加载数据。

## 约束

- 后端查询参数继续使用现有 `from` / `to`，避免改动路由和导出逻辑。
- 页面视觉继续遵守 Ocean Blue Admin 规范，不引入紫色、重阴影或风格跳变。
- 保留现有其他筛选项、分组 tab、导出按钮、分页参数传递逻辑。

## 方案

### 日期范围控件

- 用一个可点击的触发器代替原先两个可见 `type="date"` 输入框。
- 保留两个隐藏的 `<input type="hidden" name="from">` / `<input type="hidden" name="to">`，确保后端完全兼容。
- 触发器打开一个内嵌的轻量级双月历弹层：
  - 左右各显示一个月份；
  - 支持前后翻月；
  - 首次点选写入开始日期；
  - 再次点选写入结束日期；
  - 若第二次早于开始日期，则自动交换开始/结束；
  - 第二次点选完成后自动提交表单。

### 快捷范围

- 在日期范围控件旁新增四个快捷按钮：
  - 今天：`from=to=今天`
  - 昨天：`from=to=昨天`
  - 近七天：包含今天的最近 7 个自然日
  - 近一个月：包含今天的最近 30 个自然日
- 点击按钮后同步更新隐藏字段、触发器文案和面板高亮，并立即提交表单。
