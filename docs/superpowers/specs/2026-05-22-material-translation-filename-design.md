# 翻译结果素材命名规则

## 背景

素材管理的“翻译”按钮会把原始视频翻译为各小语种视频，并在回填到 `media_items` 时生成最终素材名。原逻辑使用原始素材日期和“补充素材”标记，无法直接看出这是小语种翻译结果，也无法保留原始素材日期与原负责人信息。

## 命名契约

输入示例：

```text
2026.04.01-煮蛋器-原素材-指派-陈兆阳.mp4
```

翻译结果素材名：

```text
{当前日期 YYYY.MM.DD}-煮蛋器-原素材-小语种翻译素材({小语种中文名})-20260401陈兆阳-蔡靖华.mp4
```

规则：

- 开头日期使用任务回填时的当前日期。
- 商品名使用素材库商品名。
- 标记固定为 `原素材-小语种翻译素材({小语种中文名})`。
- 来源字段由原始文件名前缀日期压缩为 `YYYYMMDD`，再拼接 `指派-` 后的原负责人。
- 末尾负责人保持 `蔡靖华`。
- 文件名任意位置都不允许出现 ASCII 空格，包括商品名后、标记中、来源字段中间和末尾。

## 兼容性

历史 `原素材-补充素材({小语种中文名})-...-蔡靖华.mp4` 命名仍保留校验兼容，避免已有素材在编辑页被误判无效。新的翻译结果命名也需要被服务端与前端校验识别为对应小语种。

## 验证

覆盖点：

- `appcore.material_filename_rules` 生成新翻译结果素材名。
- 翻译任务视频回填链路创建 `media_items.filename/display_name` 时使用新规则。
- 服务端/前端素材名校验接受新命名，同时旧命名保持兼容。
