# 数据质量提示条人话说明（2026-06-09）

## 背景

数据分析页顶部的数据质量提示条会展示订单、广告日终、广告实时、派生利润的同步状态。旧版详情使用“水位”一词，只给出“业务日 / 时间”两段信息，业务方难以判断这到底代表什么基准、哪一批同步、以及同步到什么维度。

## 需求

- 数据质量提示条继续保留默认折叠形态，避免移动端首屏被说明占满。
- 展开详情时不再使用“水位”作为展示文案。
- 每类数据基准都要用业务方能理解的字段表达：
  - 数据日期：该类数据当前覆盖到哪个 Meta 业务日或利润业务日。
  - 这批同步：最近一次订单入库、广告导入、实时快照或利润重算的时间。
  - 同步维度：说明这批数据按什么粒度同步或计算，例如店铺 + 订单行、Meta 业务日 + 广告账户 + Campaign。
- 顶部数据源 chip 不使用“兜底”等偏内部的词，改成“实时同步版 / 日终确认版 / 日终+实时混用”等直白说明。
- 不改变后端 `data_quality` 结构、状态判断、同步任务、统计口径或对账逻辑。

## 设计

共享组件 `web/templates/_data_quality_bar.html` 的 `summarizeWatermarks` 只做展示转换：

| data_quality.watermarks key | 展示标题 | 数据日期 | 这批同步 | 同步维度 |
|---|---|---|---|---|
| `orders` | 订单数据基准 | 订单业务日 | 最近一次订单入库 / 更新 | 店铺 + 订单行（按 Meta 业务日归属） |
| `meta_daily_ads` | 日终广告数据基准 | 广告业务日 | 最近一次日终广告导入 | Meta 业务日 + 广告账户 + Campaign |
| `meta_realtime_ads` | 实时广告数据基准 | 广告业务日 | 实时快照时间 | Meta 业务日 + 广告账户 + Campaign 实时快照 |
| `derived_profit` | 利润计算数据基准 | 利润业务日 | 最近一次利润重算 | 订单行 / 包裹 + 广告费分摊 |

如果后端没有提供同步批次 ID，前端不得编造批次号；用对应时间字段说明“哪一批同步”。后续如后端补充 job id，再追加展示。

## 验收

- 默认折叠状态仍只显示一行摘要和“详情”按钮。
- 展开详情后看不到“水位”二字。
- 展开详情后每类数据都包含“数据日期”“这批同步”“同步维度”三项。
- 实时大盘、订单利润、产品盈亏页面继续共用同一组件。
- 不新增后端查询或数据库访问。
