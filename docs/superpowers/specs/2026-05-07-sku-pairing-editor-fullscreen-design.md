# SKU 配对编辑框主区放大设计

日期：2026-05-07

## 背景

素材管理列表的 SKU 配对详情弹窗用于查看和人工修正 Shopify variant、店小秘 SKU、ERP 编码、采购价和商品名。当前弹窗居中显示，宽度有限，多个编码字段在输入框内被截断，运营需要横向滚动和反复定位才能确认完整 SKU。

## 行为要求

1. SKU 配对详情弹窗在桌面端应避开左侧导航，只覆盖素材管理主内容区。
2. 弹窗内容区在主内容区内四周保留约 10% 留白，视觉上占满右下角主要工作区域。
3. 弹窗高度应随视口扩展，表格区域使用剩余高度滚动，不让页脚挤压表格。
4. 表格应尽量显示完整编码，重点扩大 Shopify SKU、店小秘 SKU、店小秘商品 SKU、ERP 编码和商品名列。
5. 移动端继续沿用全屏弹窗规则，避免在小屏上额外保留桌面端 10% 留白。

## 非目标

- 不改 SKU 刷新、保存、新增手动变体的接口和数据结构。
- 不改素材管理列表表格列。
- 不做列宽拖拽、列冻结或新交互。

## 验收

- `#skuDetailMask` 桌面端样式包含避开左侧栏的主区覆盖规则。
- `.oc-sku-detail-modal` 不再受 `1280px` 内联宽度限制，应在 10% 留白内铺满可用区域。
- `.oc-sku-detail-table-wrap` 可在弹窗 body 内伸缩，表头 sticky 仍保留。
- `.oc-sku-detail-table` 的最小宽度和关键列宽比旧版更大，使编码字段优先获得更多横向空间。
