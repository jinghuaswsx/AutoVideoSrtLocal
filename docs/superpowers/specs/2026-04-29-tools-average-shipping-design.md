# 小工具平均运费设计

## 背景

工具站需要一个所有登录用户可用的“小工具”入口。第一项工具是“平均运费”，用于把从表格或文本里复制出来的人民币运费列表快速求平均值。

## 范围

- 左侧菜单新增“小工具”，所有登录用户可见。
- 新增 `/tools/` 页面，页面内使用 Tab 承载多个工具。
- 第一个 Tab 命名为“平均运费”。
- “平均运费”提供一个多行输入框，用户粘贴后自动计算，不需要点击按钮。
- 结果显示在输入框上方，使用蓝色、粗体、大字号。

## 解析和计算规则

- 每一行单独处理。
- 自动忽略没有数字的行，例如“预估运费”。
- 对有数字的行，去掉人民币符号、逗号和空格后提取第一个数值。
- 所有有效数值求和后除以有效行数。
- 结果保留一位小数，使用 JavaScript `toFixed(1)` 的四舍五入表现，例如 `81.56` 显示为 `81.6`。
- 没有有效数值时，结果显示为 `--`，并提示“粘贴运费列表后自动计算”。

## 架构

页面是纯前端工具，不写数据库，不新增 API，不上传用户输入。Flask 只负责登录态保护和模板渲染。计算函数放在模板内的页面脚本里，并通过 `window.averageShippingTool` 暴露给测试读取。

## 文件

- `web/routes/tools.py`：新增 Blueprint，提供 `/tools/` 页面。
- `web/app.py`：注册 tools Blueprint。
- `web/templates/layout.html`：左侧菜单新增“小工具”入口。
- `web/templates/tools.html`：新增 Tab 容器和平均运费工具 UI、样式与脚本。
- `tests/test_tools_routes.py`：验证普通用户可访问、菜单入口存在、页面包含 Tab 与计算脚本。
- `tests/test_tools_average_shipping_template.py`：验证模板里的计算规则与用户给定样例。

## 验证

- 运行新增测试，确认先红后绿。
- 运行相关路由测试，确认普通用户可访问。
- 对用户给出的样例计算结果应为 `41.5`。
