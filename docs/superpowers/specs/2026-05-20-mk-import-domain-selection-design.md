# 明空素材入库发布域名选择设计

- **日期**：2026-05-20
- **范围**：明空选品页“加入素材库”入库工作进程弹窗
- **依赖**：[产品链接管理弹窗](2026-05-09-product-link-management-modal.md)、[任务中心端到端流程](2026-05-16-task-center-e2e-flow-design.md)
- **实现分支**：`codex/mk-import-new-tab-buttons`

## 背景

“加入素材库”已经能在明空选品页创建或复用产品，并把英文视频素材写入素材库。产品后续推送和 Shopify ImageLocalize 会通过产品级链接管理读取启用域名；例如用户只启用 `omu.com`，该产品推送时就只应走 `omu.com`。

当前断点是：入库流程完成后，执行操作的人还不能在同一个流程里确认产品要发布到哪些域名，需要再跳到素材管理手工配置，流程不闭环。

## 目标

在“加入素材库”弹窗中增加最后一步“选择发布域名”。入库成功后，用户必须看到当前所有域名，并可以勾选或取消勾选域名；点击“确定”后，把选择同步到该产品现有的“链接管理”配置中。保存成功后再开放后续入口。

## 方案

采用方案 A：前端复用现有产品级域名管理接口，不新建接口、不新增表。

- 入库成功后，弹窗进入最后一步：`选择发布域名`。
- 前端读取：`GET /medias/api/products/<product_id>/product-link-domains`。
- 域名列表展示全部后台域名；全局停用域名展示但不可选。
- 默认勾选沿用接口返回的 `product_enabled`，即和素材管理“链接管理”弹窗保持同一套语义。
- 用户确认后提交：`POST /medias/api/products/<product_id>/product-link-domains`，body 为 `enabled_domain_ids`。
- 保存成功后，弹窗状态变为“发布域名已同步”，再显示“继续做小语种任务 / 去任务中心 / 去素材管理 / 关闭”。

## 数据流

1. 用户点击“加入素材库”。
2. 前端按现有流程调用 `POST /mk-import/video`。
3. 服务端返回 `product_id` 和 `media_item_id`。
4. 前端用 `product_id` 拉取产品域名选项。
5. 用户勾选域名并点击“确定”。
6. 前端提交 `enabled_domain_ids` 到产品链接管理接口。
7. `media_product_link_domains` 保存产品级覆盖配置。
8. 后续推送、链接检测、ImageLocalize 继续通过 `list_enabled_product_domains(product_id)` 获取该产品实际启用域名。

## 交互

弹窗步骤从当前的“准备 / 产品 / 下载 / 写入素材 / 下一步”扩展为：

1. 准备入库
2. 创建或复用产品
3. 下载明空原视频
4. 写入英文素材库
5. 选择发布域名
6. 开放下一步入口

“选择发布域名”区域只在入库成功后出现。内容包括：

- 产品名称和产品 ID。
- 域名 checkbox 列表。
- 每个域名显示启用状态；全局停用域名不可选。
- “确定”按钮。
- 保存成功或失败的状态提示。

允许用户取消所有默认域名并保存。此时该产品没有启用发布域名，后续推送逻辑按现有“无启用域名”路径处理；弹窗需要给出轻提示，避免误以为仍会自动推送到默认域名。

## 错误处理

- 入库失败：保持现有错误定位与“重新尝试”逻辑。
- 域名列表加载失败：标记“选择发布域名”为失败，展示错误详情和重试入口，不开放后续跳转按钮。
- 域名保存失败：保留用户当前勾选状态，展示接口错误，允许再次点击“确定”。
- 产品不存在或无权限：按现有接口返回错误展示，不吞掉异常。

## 验收标准

- 入库成功后弹窗会展示域名选择步骤。
- 默认勾选和素材管理“链接管理”弹窗一致。
- 用户只勾选一个域名并确认后，素材管理对应产品的“链接管理”也只显示该域名启用。
- 后续推送读取到的产品启用域名与用户在弹窗保存的选择一致。
- 保存域名前，不显示后续跳转按钮；保存成功后，底部按钮仍以新 Tab 打开。
- 域名加载或保存失败时，用户能看到明确错误并重试。

## 非目标

- 不新增域名表或产品域名表。
- 不改变推送管理、ImageLocalize、链接检测的域名解析入口。
- 不改变后台“系统设置 → 域名”管理能力。
- 不把域名选择并入 `POST /mk-import/video` 的事务；入库成功但域名保存失败时，用户可在弹窗内重试。

## 测试计划

- 路由测试：确认现有产品链接管理 GET/POST 接口仍返回和保存 `product_enabled`。
- 前端静态测试：确认明空入库弹窗包含域名加载、保存、错误、成功后开放入口逻辑。
- 单元测试：覆盖用户只选择一个域名后，`set_product_domain_enabled_ids()` 写入产品级覆盖配置。
- 回归测试：`/mk-import/check` POST 批量检查、入库按钮状态同步、新 Tab 跳转逻辑保持不变。
