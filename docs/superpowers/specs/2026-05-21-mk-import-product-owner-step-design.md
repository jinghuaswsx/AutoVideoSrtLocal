# 明空入库产品负责人步骤调整设计

- **日期**：2026-05-21
- **范围**：选品中心 / 明空选品 / 视频素材库卡片的“加入素材库”流程
- **状态**：用户已确认设计，待实施
- **锚点来源**：用户确认“去掉选择翻译员步骤，整合到创建素材流程中，放在域名选择前一步，名称改为选择产品负责人；后续任务过程中的翻译员描述也要改成产品负责人，并标识产品负责人和不同语言翻译负责人不是同一概念”

## 上位锚点

- `AGENTS.md`：文档驱动代码、任务中心流程闭环、选品/任务/素材文档锚点。
- `docs/superpowers/specs/2026-04-26-mk-import-design.md`：旧流程在点击“加入素材库”前弹“指定翻译员”小窗，后端新品用该 user_id 写入 `media_products.user_id`。
- `docs/superpowers/specs/2026-05-20-mk-import-domain-selection-design.md`：入库成功后必须先确认发布域名，再开放后续任务入口。
- `docs/superpowers/specs/2026-05-20-task-center-per-language-assignment-design.md`：任务中心支持按语言指定不同翻译负责人。
- `docs/superpowers/specs/2026-05-20-user-work-scope-translation-design.md`：明空相关指派候选来自翻译工作用户范围。

## 背景

当前 `web/templates/mk_selection.html` 中，点击“加入素材库”会先打开独立的“指定翻译员”弹窗。这个用户 ID 实际用于新品 `media_products.user_id`，也就是素材管理里的产品负责人；后续创建小语种任务时，又会用“翻译员”语义创建语言子任务。两个概念混用会误导操作人：产品负责人是产品归属，语言翻译负责人是具体语种任务的执行人，后者可以按语言分配给不同用户。

同时，5 月 20 日的域名选择设计已经把“选择发布域名”放在入库成功后的最后一步。产品负责人应属于创建素材闭环的一部分，并且必须在域名发布前确认。

## 目标

1. 点击“加入素材库”后不再先弹独立“选择翻译员”小窗。
2. 入库进度弹窗内新增步骤“选择产品负责人”，位置在“写入素材库”之后、“选择发布域名”之前。
3. 新品必须选择产品负责人后才能保存发布域名并开放后续入口。
4. 老品沿用已有 `media_products.user_id`，展示“已沿用现有产品负责人”，不要求再次选择。
5. UI、变量、提示和请求字段优先使用 `产品负责人 / product_owner_id` 语义。
6. 后续创建小语种任务的 UI 明确使用“小语种翻译负责人”或“语言翻译负责人”，并提示它可以与产品负责人不同。

## 非目标

- 不在本主流程修改“素材管理负责人候选范围”。该任务已单独交给独立 worktree agent。
- 不改变任务中心按语言独立分配能力。
- 不新增数据库字段。
- 不改变发布域名保存接口和产品链接管理语义。
- 不改变视频下载、filename 去重、素材写入、产品链接 warning 的业务逻辑。

## 交互设计

“加入素材库”进度弹窗步骤调整为：

1. 准备素材信息
2. 检查产品与链接
3. 下载明空原视频
4. 写入素材库
5. 选择产品负责人
6. 选择发布域名
7. 后续任务入口

流程规则：

- 用户点击卡片“加入素材库”后，直接打开进度弹窗并提交入库前置检查所需信息。
- 如果服务端判断为新品，弹窗展示“选择产品负责人”下拉，候选仍来自翻译工作用户范围。
- 用户确认产品负责人后，前端带 `product_owner_id` 调用入库接口创建产品和英文素材。
- 如果服务端判断为老品，弹窗展示现有产品负责人名称，并跳过负责人选择。
- 产品负责人确认或沿用完成后，才进入“选择发布域名”。
- 发布域名保存成功后，才显示“下一步：创建小语种任务 / 去任务中心 / 去素材管理 / 关闭”。

后续小语种任务弹窗文案：

- 下拉 label 从“翻译员”改为“小语种翻译负责人”。
- 创建说明中增加短提示：`产品负责人用于素材归属；小语种翻译负责人用于当前语言任务，可与产品负责人不同。`
- 如果所有勾选语言暂时仍共用一个负责人，提示文案必须写成“所选语言暂用同一翻译负责人”，避免暗示产品负责人就是翻译负责人。

## API 与兼容

`POST /mk-import/video` 增加优先字段：

```json
{
  "mk_video_metadata": {},
  "product_owner_id": 33
}
```

兼容旧字段：

```json
{
  "mk_video_metadata": {},
  "translator_id": 33
}
```

服务端解析规则：

1. 优先读取 `product_owner_id`。
2. 若缺少 `product_owner_id`，继续接受旧 `translator_id` 作为兼容别名。
3. 新品创建时使用产品负责人写入 `media_products.user_id` 和新英文素材归属。
4. 老品忽略本次传入的负责人，继续沿用已有产品负责人。
5. 校验错误文案从“invalid_translator”语义收敛为产品负责人语义；兼容错误码可以暂保留，但前端展示必须使用“产品负责人”。

## 数据语义

- 产品负责人：`media_products.user_id`。负责产品归属、素材管理负责人显示、老品入库默认归属。
- 小语种翻译负责人：任务中心子任务 `assignee_id`。负责具体语言翻译；可按语言不同而不同。
- 兼容字段 `translator_id`：仅为旧调用方兼容，不再作为新 UI 语义。

## 错误处理

- 产品负责人列表加载失败：负责人步骤标记失败，展示错误并允许重试，不进入域名选择。
- 新品未选择产品负责人：不提交入库请求，负责人步骤提示必选。
- 产品负责人无效：接口返回 400，前端提示“产品负责人不合规，请重选”。
- 入库成功但域名加载失败：沿用现有域名选择错误处理，允许重试。
- 小语种任务创建失败：继续按任务中心契约展示请求路径、HTTP 状态和后端错误原因。

## 验收标准

1. 点击“加入素材库”不再出现独立“指定翻译员”弹窗。
2. 入库进度弹窗中“选择产品负责人”位于“选择发布域名”之前。
3. 新品未确认产品负责人时不会进入发布域名保存与后续任务入口。
4. 老品显示沿用现有产品负责人，并继续复用老品归属。
5. 前端新增请求使用 `product_owner_id`，旧 `translator_id` 仅保留兼容。
6. 小语种任务弹窗不再把任务负责人泛称为“翻译员”，而使用“小语种翻译负责人 / 语言翻译负责人”。
7. 页面文案明确区分产品负责人和语言翻译负责人可以不同。
