# 原始去字幕视频素材标题校验设计

## 目标

在“素材管理”里上传原始去字幕视频素材时，要求“名称”字段符合宽松命名格式：

```text
YYYY.MM.DD-产品名-xxxxxx.mp4
```

该规则只校验展示标题，不要求实际本地文件必须重命名。前端仍默认把所选视频文件名带入名称框，用户可在提交前修改。

## 校验规则

- 日期段必须是合法日期，格式为 `YYYY.MM.DD`。
- 日期后必须紧跟英文半角连字符 `-`。
- 产品名段必须等于当前产品的 `name`，并且后面紧跟 `-`。
- 最后一段只要求非空。
- 标题必须以 `.mp4` 结尾，大小写不敏感。
- 标题会 trim 首尾空白；中间内容不限制，保持“相对宽松”。

示例：

- 合法：`2026.03.25-可堆叠棒球帽收纳盒-原始视频.mp4`
- 非法：`2026-03-25-可堆叠棒球帽收纳盒-原始视频.mp4`
- 非法：`2026.03.25-其他产品-原始视频.mp4`
- 非法：`2026.03.25-可堆叠棒球帽收纳盒-.mp4`
- 非法：`2026.03.25-可堆叠棒球帽收纳盒-原始视频.mov`

## 实现方案

新增一个专门用于 raw source 标题的校验 helper，后端上传接口在写文件前调用，返回结构化错误：

```json
{
  "error": "raw_source_title_invalid",
  "message": "原始去字幕视频素材名称不符合命名规范",
  "details": ["名称必须以 .mp4 结尾"],
  "suggested_title": "2026.04.23-产品名-原始视频.mp4"
}
```

前端 `web/static/medias.js` 在提交上传表单前使用同样的宽松规则做即时拦截，并弹窗显示错误和示例，减少等待上传后才失败的情况。后端校验保留为最终防线，避免绕过页面直接上传。

## 不做范围

- 不改数据库 schema。
- 不限制 `xxxxxx` 的字符集。
- 不要求视频文件对象 key 或本地文件名改为同名。
- 不改变原始去字幕素材的列表、翻译、删除流程。

## 验证

- 后端路由测试覆盖合法标题、错误日期、错误产品名、空尾段、非 `.mp4` 后缀。
- 现有素材上传路由测试继续通过。
- 前端手动逻辑通过代码检查确认提交前会阻断明显错误。
