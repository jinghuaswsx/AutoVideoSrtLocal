# Omni AV Sync Gemini Scorecard Design

- Date: 2026-05-13
- Module: omni-translate `av_sync_audit` report-only scorecard

## Anchors

- `AGENTS.md`: document-driven code, isolated worktree development, and verification requirements.
- `docs/superpowers/specs/2026-05-07-omni-av-sync-audit-design.md`: `audit_timeline` is the frontend display contract for ASR text, final TTS text, visual content, and diagnosis.
- `docs/superpowers/specs/2026-05-13-multi-av-sync-understand-assess-design.md`: the modern audit flow is Doubao video understanding followed by Gemini structured assessment.
- `web/templates/CLAUDE.md`: template changes must stay inside the existing task workbench structure.

## Problem

Observed task `d8aba350-231a-45f4-909a-fb4ed77b6d75` is `av_sync_audit = report_only`, but the prompt inspector shows three calls:

1. Doubao video understanding.
2. Gemini structured assessment.
3. Gemini verification.

The visible scorecard's fourth column (`问题诊断`) should not be inferred in the browser and should not depend on the old verification step. The scorecard must be generated by Gemini 3 Flash from the three visible row inputs:

- `ASR 内容`
- `正常翻译/TTS`
- `视频画面`

## Target Behavior

For Omni `report_only` mode:

- Run exactly two LLM calls:
  - `omni_av_sync.understand`: Doubao watches the final video and writes natural-language video understanding notes.
  - `omni_av_sync.assess`: Gemini 3 Flash returns the structured per-row scorecard.
- Do not call `omni_av_sync.verify`.
- Do not run safe-auto fixes or mutate sentences/audio/subtitles.
- Store debug refs only for `av_sync_audit.understand` and `av_sync_audit.assess`.

For Gemini assessment:

- Use Gemini 3 Flash (`google/gemini-3-flash-preview`) through the `omni_av_sync.assess` use case.
- The prompt payload must expose the row input as `scorecard_rows`; each row contains `asr_index`, `asr_text`, `target_text`, and `visual_observation`.
- Use JSON schema response format and require:
  - root `timeline`
  - one item per ASR row
  - each item with `asr_index`, `sync_score`, `diagnosis`, and `recommendation`
- `sync_score` is a 0-100 score from Gemini, where 100 means fully synchronized.
- `diagnosis` and `recommendation` are Chinese strings. If there is no issue, `recommendation` must be `无需调整。`.

The frontend table should display the structured Gemini output. Missing fields mean the audit should be rerun or marked as structured assessment failed; the browser must not invent scorecard data.

## UI Problem Highlighting

The scorecard must make problematic diagnosis rows visually obvious for operators reviewing the table.

- A row is considered problematic when `diagnosis_status == "issue"`, or when `sync_score < 90`, or when `recommendation` is present and not `无需调整。`.
- Problematic rows must render as red-alert cards: red border, light red background, and red text in the `问题诊断` cell.
- Non-problematic rows keep the existing neutral/normal table style.
- The highlight is frontend-only display logic; it must not mutate the audit artifact or invent diagnosis content.

## Non-Goals

- Do not change Doubao into a JSON-producing scorer.
- Do not add a third Gemini verification call for report-only mode.
- Do not backfill old tasks by editing database state.
- Do not change safe-auto behavior except where explicitly needed to keep report-only at two calls.

## Verification

- Backend tests prove Omni `report_only` produces only two debug refs: `understand` and `assess`.
- Backend tests prove `omni_av_sync.assess` uses Gemini 3 Flash and a schema requiring `sync_score`, `diagnosis`, and `recommendation`.
- Backend tests prove the Gemini assessment prompt contains `scorecard_rows` and no verification prompt is called for report-only mode.
- Frontend asset tests continue to require the scorecard table columns.
- Frontend asset tests require the problem-row predicate, red card class, and diagnosis-cell red text class.

## 2026-05-15 翻译后处理重构

将 `asr_text_zh` 和 `target_text_zh` 从 Gemini assess 步骤中分离，改为 assess 完成后的独立后处理步骤。

### 动机
- assess 的职责是音画同步评分（判断/推理），翻译是机械任务，不应耦合
- 翻译可用专用模型 `text_translate.generate`（同为 Gemini 3 Flash），与评分隔离
- 翻译失败不阻塞评分结果展示
- 重跑审计不会重复翻译

### 架构
```
assess (纯评分，不翻译) → audit_timeline 落定 → _translate_timeline_chinese 后处理
```
- `_translate_timeline_chinese` 收集需要翻译的 asr_text / target_text，一次批量调用 `text_translate.generate`
- 触发条件：源语言非中文时翻译 ASR，目标语言非中文时翻译 target
- 前端 `avSyncTargetChineseReference` 优先读取 LLM 产出的 `target_text_zh`，无数据时回退 ASR 中文原文兜底
