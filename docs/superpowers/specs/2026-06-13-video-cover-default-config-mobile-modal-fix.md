# 文案封面默认配置移动端弹窗修复

日期：2026-06-13
状态：已确认

## 背景

文案封面生成列表页 `/video-cover` 的「默认配置」弹窗包含视频分析、产品分析、文案创作、封面生成四步模型配置。移动端单列显示后，表单高度超过可视区域，底部「保存配置」按钮会被浏览器底部工具栏遮住或落到屏幕外，导致选择配置后无法保存。

## 目标

1. 「默认配置」弹窗在移动端必须自适应当前可视高度。
2. 四步配置内容区域允许纵向滚动。
3. 底部「取消 / 保存配置」操作区必须始终留在弹窗底部且可点击。
4. 弹窗底部必须预留 iOS safe area，避免被 Safari 底部工具栏遮挡。

## 非目标

- 不改变默认配置的数据结构、接口、权限或保存逻辑。
- 不改变四个步骤的供应商和模型池。
- 不改新建项目弹窗、详情页重选模型弹窗或后端执行链路。

## 实现

- `web/templates/video_cover_list.html`：
  - 默认配置弹窗使用 `100dvh` 和 `env(safe-area-inset-bottom)` 限制最大高度。
  - 表单拆成可滚动 `.vc-config-body` 和固定底部 `.vc-config-actions`。
  - 移动端按钮改为双列等宽并提高触控高度。
