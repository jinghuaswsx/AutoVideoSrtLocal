# 素材管理 ROAS 维护设计

最后更新：2026-04-28

## 目标

在素材管理列表的每个产品操作区增加“ROAS”按钮，用于维护该产品的采购、物流、尺寸和售价信息，并在弹窗中实时计算独立站保本 ROAS。

## 范围

本期先实现独立站口径：

- 采购价格
- 预估小包成本
- 实际小包成本
- 长宽高尺寸
- 独立站售价
- 预估保本 ROAS
- 实际保本 ROAS
- 当前采用的有效保本 ROAS

TK 相关字段先作为可选数据预留在弹窗靠后区域：

- 1688 采购链接
- TK 海运成本
- TK 空运成本
- TK 售卖定价
- TK 海运保本 ROAS
- TK 空运保本 ROAS

TK 计算可以复用同一手续费和成本结构，但本期不作为必填流程。

## 计算口径

手续费固定按售价的 10% 计算。

预估可用于广告的金额：

```text
独立站售价 * 0.9 - 采购价格 - 预估小包成本
```

预估保本 ROAS：

```text
独立站售价 / 预估可用于广告的金额
```

实际可用于广告的金额：

```text
独立站售价 * 0.9 - 采购价格 - 实际小包成本
```

实际保本 ROAS：

```text
独立站售价 / 实际可用于广告的金额
```

有效保本 ROAS：

- 如果填写了实际小包成本，取实际保本 ROAS。
- 如果没有填写实际小包成本，取预估保本 ROAS。
- 如果可用于广告的金额小于等于 0，显示“无法保本”，不输出误导性的 ROAS 数值。

## 数据模型

在 `media_products` 上追加 ROAS 维护字段。字段本身是产品级信息，不需要单独拆表；这样列表查询和产品详情接口都能沿用现有 `SELECT p.*`。

金额使用 `DECIMAL(10,2)`，尺寸使用 `DECIMAL(8,2)`，链接使用 `VARCHAR(2048)`。计算结果由前端实时计算，不落库存储，避免历史公式变更时数据不一致。

## 接口

复用 `PUT /medias/api/products/<pid>` 保存字段。`GET /medias/api/products` 和 `GET /medias/api/products/<pid>` 在 `_serialize_product` 中返回这些字段。

前端弹窗打开时优先使用列表行已有产品数据；如缺字段或需要最新值，可以调用产品详情接口刷新。

## 交互

素材管理列表每行操作区增加一个“ROAS”按钮。

点击后弹出独立弹窗，顶部展示：

- 产品 ID
- 产品中文名
- 产品英文名，优先使用 `product_code`
- 产品主图

下方是独立站必填区。输入时立即计算预估/实际/有效保本 ROAS。弹窗底部提供“保存”和“取消”。

TK 字段放在靠后区域，保持可选，不阻塞保存独立站数据。

## 验证

新增低耦合测试：

- 纯 Python ROAS 计算函数测试，不依赖 MySQL。
- 产品更新白名单/数值归一化测试，不连接数据库。
- 路由序列化字段测试，确保接口返回 ROAS 字段。
- 前端模板/脚本静态测试，确保 ROAS 按钮、弹窗挂载点、核心计算函数存在。

完整素材管理 DB 测试需连接测试服务器 MySQL；当前 Windows 本地按项目规则不启用本地 MySQL。
