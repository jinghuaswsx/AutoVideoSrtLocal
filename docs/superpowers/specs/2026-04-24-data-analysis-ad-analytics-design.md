# 数据分析广告分析长期导入设计

最后更新：2026-04-24

## 目标

把现有“订单分析”模块升级为“数据分析”模块，并新增“广告分析”Tab。广告分析支持长期导入 Meta 广告后台导出的 CSV/Excel 数据，按周报或月报保存历史快照，自动关联素材库产品，并在同一报表周期内关联 Shopify 订单数据。

## 范围

- 页面总名从“订单分析”改为“数据分析”。
- 保留现有订单导入、订单分析能力。
- 新增广告分析 Tab，支持上传 Meta 广告报表。
- 广告报表长期持久化，支持周报、月报和重复上传去重。
- 广告行通过广告系列名称关联 `media_products.product_code`。
- 广告分析展示广告指标和同周期 Shopify 订单指标。

## 数据模型

新增 `meta_ad_import_batches` 表记录每一次上传：

- 文件名、文件哈希、导入频率、报表开始/结束日期。
- 原始行数、导入行数、更新行数、跳过行数、匹配产品行数。
- 创建时间。

新增 `meta_ad_campaign_metrics` 表记录每个报表周期内每个广告系列的一行指标：

- 报表开始/结束日期、导入频率、广告系列名称。
- 标准化广告代码、用于匹配的产品代码、`product_id`。
- Meta 指标：花费、购物转化价值、ROAS、购买次数、链接点击、加购、发起结账、展示、CPM、CTR、CPA、平均客单价、视频平均播放时长。
- `raw_json` 保留原始行，方便后续补字段。
- 唯一键：`report_start_date + report_end_date + campaign_name`。

同一个报表周期和同一个广告系列重复上传时，更新已有行，不重复叠加。不同周期的数据作为不同快照保存，分析时用户选择具体周期查看，避免周报和月报重叠时被误合并。

## 导入与去重语义

Meta 当前导出的数据是“报表周期聚合”，不是每日明细。系统按报表快照处理：

- 周报：通常覆盖 7 天。
- 月报：通常覆盖自然月或用户导出的一个月范围。
- 重复上传同一个周期：覆盖更新对应广告系列指标。
- 上传同一文件：识别文件哈希并提示已处理，但仍允许覆盖更新，保证后台数据修正后可以重导。
- 上传部分重叠周期：保存为独立快照，分析时不跨周期相加。

## 产品关联

广告系列名称按如下规则关联素材库：

1. 小写并 trim 后，先用广告系列名称精确匹配 `media_products.product_code`。
2. 如果广告系列名称不以 `-rjc` 结尾，则再尝试追加 `-rjc` 匹配。
3. 如果广告系列名称以 `-rjc` 结尾，则再尝试移除 `-rjc` 匹配，用于兼容旧数据。

成功关联后写入 `product_id` 和实际命中的 `product_code`。未命中的广告系列仍保存并在页面上标记“未匹配”，方便后续补产品库。

## 订单关联

广告数据通过 `product_id` 与 `shopify_orders.product_id` 关联。同一个广告报表周期内，统计对应产品在 Shopify 订单中的：

- 订单数。
- 商品件数。
- 订单商品收入。

广告分析默认按所选报表周期展示。对于未匹配产品的广告行，不展示订单指标。

## 页面设计

数据分析页面包含三个 Tab：

- 订单导入：原“导入文件”能力。
- 订单分析：原“数据分析”能力。
- 广告分析：新增广告导入和广告指标分析。

广告分析 Tab 包含：

- 周报/月报选择。
- Meta CSV/Excel 上传区。
- 最新导入结果和累计统计卡片。
- 报表周期选择器。
- 产品维度分析表：产品、广告系列、花费、Meta 购买数、购物转化价值、ROAS、CPA、链接点击、加购、发起结账、展示、Shopify 订单数、Shopify 件数、Shopify 收入、匹配状态。
- 未匹配广告系列列表。

## 错误处理

- 缺少必需列时拒绝导入，并返回缺失列名。
- 日期解析失败时拒绝导入。
- 数字字段为空时按 0 或空值处理，页面展示为 `0` 或 `-`。
- 数据库写入失败返回明确错误，不吞掉异常。

## 验证

- 单元测试覆盖 Meta CSV 解析、产品代码归一化、重复周期 upsert、分析汇总查询结构。
- 路由测试覆盖页面重命名和广告分析 Tab、广告上传 API。
- 手工验证时使用测试服务器环境，不依赖本地 MySQL。
