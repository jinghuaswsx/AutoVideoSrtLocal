# 任务中心移动端列表与筛选区折叠

- 状态：已确认
- 日期：2026-06-09
- 页面：`/tasks/`
- 锚点：`AGENTS.md`、`web/templates/CLAUDE.md`、`web/static/CLAUDE.md`

## 背景

移动端查看任务中心时，筛选区会占据首屏较大空间；任务列表表格本身有较宽列宽，但移动端横向滚动容器不够稳定，导致用户看不到完整数据。用户需要在看数据时能收起搜索框、任务类型、紧急状态、负责人、任务状态等功能区，后续需要筛选时再展开。

## 目标

1. 任务中心移动端必须能看到任务列表数据，表格保持原有表格结构，通过横向滚动查看完整列。
2. 筛选功能区提供“收起筛选 / 展开筛选”按钮，隐藏或恢复搜索框、筛选框、刷新按钮和筛选统计。
3. 折叠状态只影响前端展示，不清空已有筛选值，不改变列表接口参数。
4. 桌面端保留现有高密度筛选布局；移动端筛选控件宽度适配单列或双列，不挤出屏幕。

## 实现约束

- `web/templates/tasks_list.html` 继续作为任务中心页面模板，不拆分新路由。
- 表格不改卡片化，不隐藏表头；移动端使用 `.tc-table-scroll` 横向滚动，并禁用移动端 sticky 表头。
- 折叠按钮必须可通过 `aria-expanded` 表示当前状态，切换文案与状态保持一致。
- 切换大 TAB 时，非任务总览区域继续隐藏任务筛选功能区。

## 非目标

- 不改任务中心 API、任务状态机、权限过滤、分页数量或排序规则。
- 不新增数据库字段、迁移或后端服务。
- 不改任务详情抽屉的业务流程。
