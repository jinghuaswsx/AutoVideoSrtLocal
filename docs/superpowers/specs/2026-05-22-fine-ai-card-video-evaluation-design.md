# 精细 AI 评估的视频卡片素材输入

最后更新：2026-05-22

## 背景

选品中心的视频卡片里，`精细AI评估` 用于判断某个还未入库的商品视频素材是否值得加入素材库继续运营。这个入口不是只评估商品链接，也不是依赖素材已入库；卡片上的商品链接和当前视频素材共同构成评估输入。

## 规则

- 从视频卡片发起的 `精细AI评估`，未入库商品必须同时具备明文商品链接和当前卡片视频路径。
- 前端请求必须把当前卡片的视频路径、播放 URL、文件名、时长传给 `/xuanpin/api/fine-ai-evaluation`。
- 后端收到外部商品链接评估请求后，必须在服务端缓存当前卡片视频，并把缓存后的 object key 写入评估 run 的 `asset_snapshot.videos`。
- 发送给大模型前，当前卡片视频必须走现有 LLM 视频优化链路：截取 15 秒，并通过 `SHORT_CLIP_AUDIO` 转为最高 480P、15fps、600k 视频码率。
- 国家重跑时必须复用原 run 中的卡片视频 `asset_snapshot`，不能因为 `product_id=0` 重新查本地素材库而丢失视频。

## 非目标

- 不要求商品先加入素材库。
- 不改变普通 `AI评估` 的 30 秒视频评估逻辑。
- 不改变已入库商品的精细 AI 评估入口；已入库商品仍可继续使用产品素材快照。
