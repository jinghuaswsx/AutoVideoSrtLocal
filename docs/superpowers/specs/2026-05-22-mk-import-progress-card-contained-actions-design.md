# 明空入库流程卡片内操作修正设计

- **日期**：2026-05-22
- **范围**：选品中心 / 明空选品 / 视频素材库卡片的“加入素材库”进度弹窗
- **状态**：用户已确认
- **锚点来源**：用户确认“卡片应该有严格操作逻辑顺序；对应卡片相关操作全部放到卡片里面；选择产品负责人和选择发布域名都直接在对应卡片内选择并确认；后续跳转按钮放到后续任务入口卡片内；不要限制卡片高度大小”

## 上位锚点

- `AGENTS.md`：文档驱动代码、任务中心流程闭环、选品/任务/素材文档锚点。
- `docs/superpowers/specs/2026-05-20-mk-import-progress-modal-design.md`：入库弹窗必须展示当前步骤、失败原因和下一步入口。
- `docs/superpowers/specs/2026-05-20-mk-import-domain-selection-design.md`：发布域名确认成功后才能开放后续入口。
- `docs/superpowers/specs/2026-05-21-mk-import-product-owner-step-design.md`：产品负责人语义来自该设计；本修正进一步要求视觉步骤顺序必须匹配实际执行顺序。

## 背景

当前进度弹窗的操作控件已经放入对应卡片，但“选择产品负责人”仍排在“写入素材库”之后。实际执行时，前端会先让用户选择或沿用产品负责人，再调用入库接口检查产品、下载视频和写入素材库。视觉卡片顺序与真实业务顺序不一致，用户会误以为负责人是写入后的后置动作。

## 目标

1. 每个步骤卡片保持严格业务顺序：准备素材信息 → 选择产品负责人 → 检查产品与链接 → 下载明空原视频 → 写入素材库 → 选择发布域名 → 后续任务入口。
2. 产品负责人下拉框、重新加载和确认按钮只出现在“选择产品负责人”卡片内。
3. 产品负责人卡片不再出现重复说明和取消按钮；保留卡片标题、状态说明、下拉框和确认按钮。
4. 发布域名列表、重新加载和确认按钮只出现在“选择发布域名”卡片内。
5. “下一步：创建小语种任务 / 去任务中心 / 去素材管理 / 关闭”只出现在“后续任务入口”卡片内。
6. 步骤卡片不设置固定高度或最大高度，内容由下拉框、域名列表、提示和按钮自然撑开；弹窗本身可以保留视口滚动。

## 非目标

- 不改变 `POST /mk-import/video`、产品域名 GET/POST、任务中心创建接口。
- 不改变产品负责人候选范围。
- 不改变产品去重、视频下载、素材写入或发布域名保存语义。
- 不重做弹窗视觉系统，只调整操作归属和局部布局。

## 交互规则

- 弹窗打开后，步骤列表一次性渲染，所有可交互区域都挂在对应步骤卡片的内容区。
- 新品入库时，第 2 张“选择产品负责人”卡片展开下拉框；确认后该卡片标记完成，再进入第 3 张“检查产品与链接”。
- 老品入库时，第 2 张“选择产品负责人”卡片显示沿用已有负责人并保持无额外操作，然后进入第 3 张“检查产品与链接”。
- 入库成功后，“选择发布域名”卡片展开域名 checkbox 列表；保存成功前不显示后续入口按钮。
- 发布域名保存成功后，“后续任务入口”卡片展开后续按钮。
- 如果入库失败，通用“重新尝试”按钮显示在失败步骤对应卡片内。

## 验收标准

1. 页面模板中不存在进度弹窗底部独立 `mki-progress-actions` 操作区。
2. `MKI_IMPORT_STEPS` 中 `productOwner` 必须排在 `product` 前面。
3. 产品负责人控件在 `productOwner` 步骤 action 区内，且没有 `mkiImportProgressCancelProductOwner` 取消入口。
4. 发布域名控件在 `domains` 步骤 action 区内。
5. 后续入口按钮在 `next` 步骤 action 区内。
6. 相关静态测试覆盖卡片内操作归属、顺序和取消按钮移除。
7. 聚焦路由测试、模板静态测试、compileall 和 diff 检查通过。
