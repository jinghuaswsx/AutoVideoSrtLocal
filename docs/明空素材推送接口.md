# 明空素材推送接口

提供给**明空推送端**使用的一组 OpenAPI，用于拉取素材清单、按三元组精确取单条素材的推送 payload、以及回写推送结果。

全部接口以 `X-API-Key` 作为认证方式，密钥对应服务端 `.env` 的 `OPENAPI_MEDIA_API_KEY`（与素材信息查询接口共用同一把密钥）。

---

## 基础约定

| 项 | 值 |
| --- | --- |
| Base URL | `http://14.103.220.208:8888`（生产） |
| 认证请求头 | `X-API-Key: <your-api-key>` |
| 返回格式 | `application/json`（UTF-8） |
| 错误响应 | `{"error": "<slug>", ...}` |
| 下载链接 | 视频 / 封面均为 TOS **临时签名地址**，默认有效期见 `expires_in` 字段 |

统一错误状态码：

| 状态码 | 场景 |
| --- | --- |
| 400 | 请求参数缺失或非法（例如 `missing params`、`not_ready`、`link_not_adapted`） |
| 401 | `X-API-Key` 缺失或不匹配 |
| 404 | 素材 / 产品不存在 |
| 409 | 素材已推送（`already_pushed`） |

---

## 1. 获取素材推送 payload（按三元组）

推送端推送**某一条**视频时调用。三元组 `(product_id, lang, filename)` 精确定位素材，避开同产品+同语种下多条视频的歧义。

```
GET /openapi/push-items/by-keys?product_id=<int>&lang=<code>&filename=<str>
```

### Query 参数

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `product_id` | int | 是 | 产品主键 ID（不是 product_code） |
| `lang` | string | 是 | 语种代码，例如 `en` / `de` / `fr` / `es` / `it` / `ja` / `pt` |
| `filename` | string | 是 | 视频素材的原始文件名（`media_items.filename`），区分大小写 |

### 请求示例

```bash
curl -H "X-API-Key: <your-api-key>" \
  "http://14.103.220.208:8888/openapi/push-items/by-keys?product_id=123&lang=de&filename=demo.mp4"
```

### 字段说明

- `item_id`：定位到的 `media_items.id`，后续调 `mark-pushed` / `mark-failed` 时需要传这个值。
- `item.status`：`not_ready` / `pending` / `pushed` / `failed`。只有 `pending` 或 `failed` 才应该被推送；推送端通常应在推送前校验。
- `item.readiness`：4 项布尔，分别表示视频文件、视频封面、对应语种的文案、产品适配该语种。
- `payload`：可直接作为请求体 POST 给下游渠道。当前字段含义：
  - `product_links`：除英语外所有启用语种的产品站链接。
  - `videos`：**单条视频**（由三元组精确定位，数组恒定一项）。
  - `author` / `push_admin`：固定 `蔡靖华`。
  - `level`：取自 `media_products.importance`，缺省 3。

### 错误情形

| HTTP | `error` | 说明 |
| --- | --- | --- |
| 400 | `missing params` | 三元组缺一或为空，返回体带 `required` 字段 |
| 401 | `invalid api key` | 鉴权失败 |
| 404 | `item not found` | 三元组命中不到有效素材（含已软删） |
| 404 | `product not found` | 产品不存在或已软删 |

## 2. 推送端素材列表

按素材 × 语种维度的扁平列表，供推送端决定推哪些。

```
GET /openapi/push-items
```

### Query 参数

| 参数 | 类型 | 默认 | 说明 |
| --- | --- | --- | --- |
| `page` | int | 1 | 页码 |
| `page_size` | int | 20 | 每页大小，最大 100 |
| `q` | string | — | 按产品名 / product_code 模糊筛选 |
| `status` | string | — | 状态过滤，多个逗号分隔：`not_ready` / `pending` / `pushed` / `failed` |
| `lang` | string | — | 语种过滤，多个逗号分隔 |

### 响应

```json
{
  "items": [
    {
      "item_id": 456,
      "product_id": 123,
      "product_code": "sonic-lens-refresher",
      "product_name": "Sonic Lens Refresher",
      "lang": "de",
      "filename": "demo.mp4",
      "display_name": "demo.mp4",
      "file_size": 12345678,
      "duration_seconds": 28.6,
      "cover_url": "https://signed.example.com/...",
      "status": "pending",
      "readiness": { "has_object": true, "has_cover": true, "has_copywriting": true, "lang_supported": true },
      "pushed_at": null,

    }

  "total": 127,

```

## 3. 单条素材详情（按 item_id）

```
GET /openapi/push-items/<item_id>
```

返回结构与列表 `items[*]` 相同。推送端通常用不到此接口；优先使用 **三元组接口** 以避免缓存 item_id。

---

## 4. 回写推送成功

推送端向目标渠道推送成功后调用，服务端写入 `media_push_logs` 并更新 `media_items.pushed_at` / `latest_push_id`。

```
POST /openapi/push-items/<item_id>/mark-pushed
Content-Type: application/json
X-API-Key: <your-api-key>

{
  "request_payload": { "...": "本次推送使用的 payload" },
  "response_body":   "下游渠道返回的原文"
}
```

- `request_payload`：可选，会保存在日志供回溯。
- `response_body`：可选，纯文本 / JSON 字符串。

### 响应

```json
{ "ok": true, "log_id": 789 }
```

---

## 5. 回写推送失败

```
POST /openapi/push-items/<item_id>/mark-failed
Content-Type: application/json
X-API-Key: <your-api-key>

{
  "request_payload": { "...": "本次推送使用的 payload" },
  "response_body":   "下游渠道返回的原文（可空）",
  "error_message":   "网络超时 / HTTP 500 / 其他简述"
}
```

调用后：素材 `latest_push_id` 指向本条失败日志；`pushed_at` 保持为空；下次列表中该素材状态为 `failed`，推送端可据此决定是否重试。

### 响应

```json
{ "ok": true, "log_id": 790 }
```

---

## 6. 产品清单（供选型）

```
GET /openapi/materials?page=1&page_size=20&q=&archived=0
```

返回产品列表，含每个产品的主图语种、文案语种、素材语种统计。`archived` 支持 `0`（默认，未归档）、`1`（归档）、`all`（全部）。

---

## 7. 单个产品完整素材

```
GET /openapi/materials/<product_code>
```

返回产品基础信息、多语种主图 / 文案 / 素材（含 TOS 临时签名下载地址）。格式与语义详见 [素材信息获取接口 API.md](素材信息获取接口API.md)。

---

## 推荐调用流程

1. **定时拉清单**：`GET /openapi/push-items?status=pending`。
2. **取单条 payload**：`GET /openapi/push-items/by-keys?product_id=&lang=&filename=`（**本文档主推**，精确定位）。
3. **下发到目标渠道**。
4. **成功**：`POST /openapi/push-items/<item_id>/mark-pushed`。
5. **失败**：`POST /openapi/push-items/<item_id>/mark-failed`，视业务策略择机重试（重试时重新走第 2 步即可，payload 每次都会即时重新签名）。

---

## 注意事项

1. **临时签名链接过期**：payload 里的 `videos[*].url` / `image_url` 是 TOS 临时签名地址，时效见 `expires_in`（默认 3600s）。不要缓存，使用时应即取即用。
2. **推送状态一致性**：同一 `item_id` 调用 `mark-pushed` 后再次调用 `by-keys` 将返回 HTTP 200 + `status=pushed`，需要再次推送请联系管理员或后台「重置」后重试。
3. **三元组唯一性**：`(product_id, lang, filename)` 目前无 UNIQUE 约束，如存在同名素材（理论上不应出现），接口取 `id DESC` 的最新一条。已为该三元组建联合索引 `idx_product_lang_filename`。
4. **鉴权**：`X-API-Key` 必须与服务端完全一致，建议在请求方配置中心 / 环境变量中保存，不要写进代码仓库。
